# 🎓 CSAD Automated Repository Checker

Ця система автоматизує перевірку наявності репозиторіїв студентів на GitHub. Вона зв'язує **Google Таблицю** з **GitHub Actions**, дозволяючи  бачити статус здачі лабораторних ("OK" або "FAIL") прямо в таблиці, не відкриваючи кожне посилання вручну.

---

## 🚀 Як це працює (Workflow)

1.  **Експорт:** Ви натискаєте кнопку в Google Таблиці. Скрипт збирає дані (ніки та назви репо) і відправляє файл `groups_XXX.csv` у папку `input/` цього репозиторію.
2.  **Обробка:** GitHub автоматично помічає новий файл і запускає Python-скрипт.
3.  **Перевірка:** Python-скрипт:
    * Знаходить потрібні колонки (нікнейм та номер групи/назву репо).
    * Перевіряє, чи існує посилання `github.com/user/repo`.
    * Ставить статус: `OK` (є), `FAIL` (немає/прихований) або `EMPTY` (дані відсутні).
4.  **Збереження:** Результат записується у папку `output/` і завантажується назад у репозиторій.
5.  **Імпорт:** Ви натискаєте кнопку "Імпортувати" в Таблиці, і оцінки з'являються у відповідній колонці.

---

## 🛠 Налаштування

### 1. У Google Таблиці (Apps Script)
У редактор скриптів (`Extensions` -> `Apps Script`) вставлено код, який з'єднується з GitHub API.
**Важливі налаштування в `CONFIG`:**
* `repoName`: Назва цього репозиторію (наприклад, `csad-check-new`).
* `inputPath` / `outputPath`: Назви файлів. **Рекомендується використовувати номер групи**, наприклад `input/groups_401.csv`, щоб групи не перезаписували одна одну.

### 2. У цьому репозиторії
Структура файлів:
* `checker.py` — Головний скрипт перевірки. Він "всеїдний" і сам шукає колонки за ключовими словами (`git`, `repo`, `401`, `402` тощо).
* `.github/workflows/check.yml` — Інструкція для автоматизації. Налаштована на автозапуск при зміні файлів у `input/**`.

## 📖 Інструкція з використання

### Крок 1: Підготовка даних
У Google Таблиці мають бути заповнені колонки:
* **Git Name** (нікнейм на GitHub).
* **Номер групи** або **Repo Name** (назва репозиторію, наприклад `csad-task1`).

### Крок 2: Запуск перевірки
1.  У меню таблиці натисніть: **CSAD Tools** -> **1. Експортувати (На GitHub)**.
2.  Зачекайте 1-2 хвилини.
    * Можна перевірити статус у вкладці **Actions**
    * ВАЖЛИВА ПРИМІТКА на GitHub якщо Output не згенерується то в репці в секції Actions зліва тиснете на Check Repos->Run workflow->Run workflow.
    * Коли кружечок стане **зеленим** ✅ — перевірка завершена.

### Крок 3: Отримання результатів
1.  У меню таблиці натисніть: **CSAD Tools** -> **2. Імпортувати (В Таблицю)**.
2.  Скрипт оновить колонку **Status**.

## ⚡ Вирішення проблем (Troubleshooting)

### 🔴 Статус "FAIL", хоча репозиторій є
* Перевірте, чи репозиторій **Public** (публічний). Скрипт не бачить приватних репозиторіїв без доступу.
* Перевірте правильність написання ніку та назви репо (без зайвих пробілів).

### 🟡 Скрипт на GitHub не запускається
* Перейдіть у вкладку **Actions**.
* Виберіть `Check Repos` зліва.
* Натисніть кнопку **Run workflow** (справа), щоб запустити перевірку примусово.

### 🟠 Помилка при Експорті/Імпорті в Таблиці
* Переконайтеся, що в налаштуваннях `CONFIG` (Apps Script) вказана правильна назва репозиторію та токен.
* Якщо виникла помилка `git push rejected` в логах GitHub — скрипт автоматично спробує виправити це через `git pull --rebase`. Просто запустіть Експорт ще раз.
